From C to CPP

macro

常量宏

在编译的预处理阶段进行简单的字符替换，为程序提供更好的可读性，避免魔法数

C语言的NULL就是语言预定义的宏

函数宏

函数宏的初衷是消除函数调用的开销，保持函数的结构。在运行时，不会带来额外的时间和空间开销，而函数会在运行时执行压栈出栈的操作，存在函数调用的开销。

但是建议写函数，由于函数宏可读性差，可能出现奇怪的bug

控制宏

用于防止头文件重复包含，实现一种代码开关控制

规范的头文件的结构都应该是如下结构：

头文件

头文件应该含有如下内容：

  • macro
  • function declearation (different with definition in the .cpp file )
  • struct

头文件只能被包含一次

struct

sizeof()是一个操作符，结构体的sizeof涉及到字节对齐的问题

字节对齐的细节和编译器的实现相关，但一般而言，满足三个准则：

  • 结构体变量的首地址能够被其最宽基本类型成员的大小所整除。
  • 结构体的每个成员相对于结构体首地址的偏移量（offset）都是成员大小的整数倍，如有需要，编译器会在成员之间加上填充字节（internal adding）。
  • 结构体的总大小为结构体最宽基本类型成员大小的整数倍，如有需要，编译器会在最末一个成员后加上填充字节（trailing padding）。

可以实现强制按字节对齐，网络通信节约带宽，降低性能

run time memory

stack

  • parameter and local variables of the function
  • compiler allocate and free
  • Don’t return address of a local variable.
  • contiguous memory usage

heap

  • new allocate memory in this section
  • programmer allocate and free. If not free, OS will free.
  • discrete memory usage

static

global variables,const variables static variables and string

code

  • machine commands for the cpu
  • shared and read-only

A better C

Overloading

  • same function name, parameters differ.

  • different types, numbers and orders.

  • can’t use return type to overload, maybe the return value is not important at all.

    add c source file in c++ project

Default parameter

  • add parameters when comiling
  • must stay in the end
  • declaration and function can’t both have default parameters. (Which one to follow?)

占位参数

  • 初衷是高瞻远瞩，为了程序的迭代，未实现的方法；

  • 实际不常用，用途主要是强制重载

  • c语言也可以在声明时使用占位参数，但不能在定义时使用

Encapsulation

class

  • how to display data (data member)
  • how to manage data (method)
  • . is a new operator

value：essential elements

address：not necessary

access control

  • public: everyone
  • protected: self, friend and derived class
  • private (default) : self and friend

友元：

定义在类外部，但有权访问类的所有私有（private）成员和保护（protected）成员，可以是友元函数也可以是友元类

constrctor

  • 与类同名
  • 任意参数
  • 无返回值
  • 创建类的新对象时调用

初始化列表

作用：初始化const成员变量；对成员变量进行赋值；协调父类构造函数

  • 成员变量的初始化顺序与初始化列表中列出的变量的顺序无关，它只与成员变量在类中声明的顺序有关（在栈上分配内存）

  • 初始化 const 成员变量的唯一方法就是使用初始化列表

default constructor

什么都不传，初始化虚指针，程序员没有写构造函数时默认存在；如果实现了任意一个构造函数则被替代，不再存在

拷贝构造

实现对象的复制

只有一个参数，本类型的引用（const classname &obj）

  • 通过使用另一个同类型的对象来初始化新创建的对象。
  • 复制对象把它作为参数传递给函数。
  • 复制对象，并从函数返回这个对象。

如果没有定义，编译器会自动定义一个

默认是浅拷贝，自行实现的是深拷贝

destructor

  • ~与类同名
  • 每次删除创建的对象时执行
  • 没有返回值
  • 没有参数
  • 唯一，除非程序员已经实现，编译器会自动生成

具体来看执行的时候。有以下几种情况：

  • 程序完成执行
  • 局部变量作用域结束 {}
  • delete opertaor
